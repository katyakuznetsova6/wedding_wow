# Музыка для сайта

## Почему на GitHub Pages не играло с Pixabay

Внешние ссылки на трек часто **не работают** в продакшене: редиректы, срок действия URL, блокировки по `Referer`, политика CDN. Надёжный вариант — **файл в репозитории**.

## Что сделать

1. Экспортируйте трек в **MP3** (права на использование — ваши или лицензия).
2. Сохраните файл с именем **`music.mp3`** в эту папку:  
   `assets/audio/music.mp3`
3. Закоммитьте и запушьте в ветку, с которой публикуется GitHub Pages.

В `index.html` уже указано:

```html
<source src="assets/audio/music.mp3" type="audio/mpeg">
```

Если хотите другое имя — переименуйте файл и обновите `src` в `index.html`.

## GitHub Pages

- Репозиторий → **Settings** → **Pages** → выберите ветку (часто `main`) и папку **`/ (root)`**, если `index.html` в корне репо.
- После деплоя откройте сайт по адресу вида `https://<user>.github.io/<repo>/` — путь к аудио будет `.../<repo>/assets/audio/music.mp3` (относительные ссылки из `index.html` подстроятся сами).

## Размер и формат

- Лучше **до ~5–8 МБ** для быстрой загрузки.
- Формат **MP3**, `type="audio/mpeg"` в разметке.

## Запасной вариант

Второй `<source>` в `index.html` — демо-трек с SoundHelix, если локального файла ещё нет (удобно для проверки верстки). После добавления своего `music.mp3` браузер обычно берёт **первый** успешно загруженный источник.

## Если тишина после деплоя

1. Откройте в браузере напрямую: `https://ваш-сайт.github.io/имя-репо/assets/audio/music.mp3` — должен начаться скачивание или проигрывание. Если **404** — файл не попал в репозиторий или неправильное имя/папка.
2. В **консоли** (F12 → Network) при клике «Открыть приглашение» посмотрите запрос к `music.mp3`: статус должен быть **200**.
3. Музыка включается **только после клика** «Открыть приглашение» (так устроены браузеры) — автозвук без жеста пользователя блокируется.
4. Ссылки на **Pixabay и др. CDN** с главной страницы GitHub часто ломаются — поэтому локальный `music.mp3` в репо предпочтительнее.
