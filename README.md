# Свадебное приглашение — Алексей и Екатерина

Одностраничный сайт-приглашение: приветственный экран, анимация на первом блоке, обратный отсчёт до события, локация и такси, тайминг, дресс-код, RSVP с сохранением в браузере, контакты организатора.

## Возможности

- **Приветствие** — модальное окно при первом визите в этой сессии (`sessionStorage`), два варианта входа: с фоновой музыкой или без.
- **Первый экран** — смена текста и фотографий, кнопка прокрутки к следующему блоку.
- **Таймер** — дни, часы, минуты, секунды до `25.08.2026 16:00` (Челябинск, UTC+5).
- **RSVP** — имя, присутствие, напитки; данные в `localStorage` (без сервера).
- **Музыка** — плавающая кнопка ♪ после входа на сайт.

## Структура репозитория

```
├── index.html          # Разметка и контент
├── styles.css          # Стили
├── app.js              # Логика
├── assets/images/      # Ваши фото для фонов (см. assets/images/README.md)
├── package.json        # Скрипт локального превью
└── README.md
```

## Запуск локально

Требуется Node.js (для `npx`):

```bash
npm start
```

Откройте в браузере адрес из вывода (по умолчанию порт **5173**).

Без Node можно открыть `index.html` напрямую в браузере — пути к картинкам относительные (`assets/images/...`).

## Публикация

Залейте содержимое папки на любой статический хостинг (GitHub Pages, Netlify, Vercel, Timeweb и т.д.). Убедитесь, что папка `assets/images/` попала в деплой.

## Что поменять под себя

- Телефон и Telegram в блоке контактов (`index.html`).
- Дата/время события в `app.js` (`new Date("2026-08-25T16:00:00+05:00")`).
- Ссылка на трек в `<audio>` или замена на свой файл в `assets/`.
- Тексты и фото в `index.html` / `assets/images/`.
